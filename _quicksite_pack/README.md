# Optimacy Quick Site Plan MVP - Codex Pack

Purpose: create a separate, stripped-down MVP repository using the current Land Portal codebase as a donor, not as a coupled front door.

Target product: a simple address-to-site-plan workflow:

1. Search address
2. Select parcel
3. View aerial/streets/topo map
4. Toggle parcel/adjoining/contour/building/reference layers
5. Draw basic site-plan objects
6. Add dimensions/labels
7. Export a planning-exhibit PDF

This pack includes:

- `docs/reuse-audit.md` - what to reuse, adapt, or leave behind
- `docs/repo-setup.md` - how to create the new GitHub repository safely
- `docs/sprint-plan.md` - 4-sprint MVP plan
- `docs/component-plan.md` - proposed simplified component tree
- `codex/*.md` - Codex-ready patch prompts/specs
- `scaffold/` - starter file structure and representative components for the new repo

Recommended repo name: `optimacy-quicksite` or `optimacy-siteplan-mvp`.
