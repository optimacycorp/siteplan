# Feedback and Current State

## What is working well
- title commitment import has become a real workflow
- linked child documents exist as separate records
- preview and document hierarchy are present
- the product direction now clearly includes title + parcel + survey + design

## Main gaps
1. The system is still only partially parcel-first.
2. The designer still lacks full map-backed spatial context.
3. Too much orchestration logic still lives in UI/page files.
4. Survey and constraints are not yet driving design decisions.

## Recommendation
The next work should focus on:
1. finishing title hardening
2. making parcel selection drive the map/designer
3. adding adjoining parcels in the designer
4. connecting survey compare to parcel geometry
5. generating buildable-envelope constraints for design
