# Target State

## Product target
The product should operate as:

Parcel -> Title -> Survey -> Constraints -> Design -> Output

## Core outcomes
- one active parcel anchor per project
- map auto-fits to the selected parcel
- designer shows selected parcel plus adjoining parcels
- survey compare overlays reconstructed/title geometry against parcel geometry
- buildable envelope becomes the basis for design
- title/import flows are stable and readable
